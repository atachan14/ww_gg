# 要件整理

## 概要
人狼ゲームにおける確率計算アルゴリズムを作る。
ユーザーが公開情報として見えている盤面を入力し、それに応じた各行動の期待勝率を表示する。

## 計算結果の例
### 前提
市民1 / 狼1 の盤面は狼の勝利とする。
このときの勝率は、白陣営 0% / 黒陣営 100% である。

### 夜 市民2 / 狼1 の盤面
狼は自分以外を噛めば勝利するため、その行動の黒陣営勝率は 100% となる。
自分を噛んだ場合は敗北するため、その行動の黒陣営勝率は 0% となる。
狼が「もっとも勝率の高い噛み先を選択する」戦略を取るなら、この盤面での黒陣営勝率は 100% である。

### 昼 市民2 / 狼1 の盤面
村視点では全員が同じ立場に見えるため、誰を処刑しても同じ期待値になる。
誰を処刑しても白陣営勝率は 33%、黒陣営勝率は 66% である。

### 昼 市民5 / 狼1 の盤面
村は誰を処刑しても 20% の確率で勝利する。
残り 80% では、夜 市民3 / 狼1 の盤面を経由して、次の昼に 市民2 / 狼1 の盤面へ進む。
その盤面での白陣営勝率は 33% なので、全体の白陣営勝率は `20% + 80% * 33%` で約 46.7% となる。

### 昼 レギュレーション: 占1 / 市民3 / 狼1、公開盤面: Day1昼 占2 / 市民3
村が「村騙りをしない」戦略を取る場合、2人の占い師候補はそれぞれ 真 50% / 偽 50% となる。
一方で 3人の市民候補はそれぞれ 真 100% / 偽 0% となる。
占い結果に関わらず占い師候補を処刑すると 50% の確率で勝利する。
勝利しなかった場合は、処刑された占い師候補が 真 100% / 偽 0% だったことが確定し、Day1夜 占1 / 市民3 を経由して Day2昼 占1 / 市民2 の盤面になる。
その時点では残っている占い師候補を処刑すれば勝利するため、Day1昼時点の白陣営勝率は 100% となる。

### 昼 レギュレーション: 占1 / 市民3 / 狼1、公開盤面: Day1昼 占1 / 市民4
村が「対抗が出なければ真として認める」戦略を取る場合、占い師候補は 真 100% / 偽 0% となる。
占い師はゲーム開始時にランダム白を 1 つ保有するため、その白先の市民も 真 100% / 偽 0% となる。
残った市民 3 人はそれぞれ 真 33% / 偽 66% となるため、Day1昼にその中の誰かを処刑すると 33% で勝利する。
勝利しなかった場合は、処刑された市民が 真 100% / 偽 0% であることが確定し、Day1夜 占1 / 白を貰った市民1 / 未占い市民2 の盤面へ進む。
狼が未占い市民を噛んだ場合、Day2昼には狼位置が確定し、白陣営が最適行動を選ぶなら狼は敗北する。
狼が白を貰った市民を噛んだ場合も、最終的に狼は敗北する。
狼が占い師を噛んだ場合は、Day2昼 白を貰った市民1 / 未占い市民2 の盤面となり、白陣営と黒陣営の勝率はそれぞれ 50% となる。
したがって Day1昼時点の白陣営勝率は `33% + 50%` で 83% となる。

## 画面構成
### TOP (`/top`)
- サイトタイトル
- レギュレーションキャストの入力
- レギュレーションルールの選択
- 各陣営や役職の戦略設定

#### レギュレーションルール
レギュレーションキャストに応じてタブを追加する。
狼
- 狼の自噛: 有 / 無
- 狼の無噛: 有 / 無

占
- 初日の占い結果: ランダム白 / 有（選択して占い=黒が出る可能性がある） / 無

狩
- 同一対象の連続護衛: 有 / 無


### NodeScreen (`/day{X}_{phase}`)
#### ヘッダー
- 画面左上に `Day` と `Phase` を表示
- 画面中央上に対象 Node の白陣営勝率 / 黒陣営勝率を表示

#### メインテーブル
公開されている盤面情報の入力と、それに応じた各期待値を表示する。

カラム案:
`index | 名前 | 生死 | CO | 白陣営% | 黒陣営% | 選択した場合の白陣営勝率 | 選択した場合の黒陣営勝率 | ラジオボタン`

補足:
- `白陣営%` は子カラムの合計値
- `黒陣営%` は子カラムの合計値
- レギュレーションに応じて、白陣営側には `占% / 霊% / 狩% / 市民%` を、黒陣営側には `狂% / 狼%` を子カラムとして表示する
- CO された役職に応じて、2段目以降に能力結果入力欄を表示する

#### 戦略オプション
- メインテーブルの下に配置する
- 陣営や役職ごとにタブ化する

#### Tree 一覧
- メインテーブルのラジオボタンで対象を選択したうえで `Tree` ボタンを押すと、画面右側に以降の Tree 一覧を表示する
- Tree には、戦略オプションどおりに進行した場合の branch と、各 node ごとの勝率を表示する
- 勝率は「その node 時点の瞬間値」と「そこから先を含めた合計値」の 2 種類を扱う

## 画面遷移
Day 画面では、ラジオボタンで対象を選択してから次の操作を行う。

- `Tree` ボタン: ページ遷移せず、画面右側に Tree を表示する
- `進む` ボタン: 選択中の Tree における次の Node へ進む
- `戻る` ボタン: 前の Node へ戻る。Day1 昼では非表示とする

## 将来的な Phase 構造
1. 昼: 処刑対象の選択
2. 占: 占い師候補ごとに占い対象を選択
3. 狩: 狩人候補ごとに護衛対象を選択
4. 夜: 噛み対象の選択

## 命名の基本方針
- `Phase`: 昼 / 夜 / 能力選択などの時間区分
- `Node`: Tree の各要素
- `NodeState`: 各 Node が持つ盤面情報
- `Action`: 処刑 / 噛み などの分岐要因

## 陣営の分類
- 白陣営: 村 / 占 / 霊 / 狩
- 黒陣営: 狂 / 狼

## 計算方針
- レギュレーションと盤面の差異を確認し、人数が過剰な位置に狼の数をカウントする
- 各プレイヤーを処刑した場合にゲームが終了する可能性を保存し、ゲームが 100% 終了するまで各 Day の Tree を辿る
- 盤面上で同じ立場のプレイヤーは同等の計算結果として扱い、計算を省略する

### `期待勝率最大を選ぶ` が同値だった場合の扱い
- メインテーブルには最大勝率のみを表示する
- 誰を選択したかが不要な場面では、同値なら誰を選んでもよい
- アルゴリズム上で代表値が必要なら、`index` が小さい方を採用する
- Tree 一覧でも同様の扱いとする
