# v1 仕様

## 目的
最初の実装では、能力結果の入力や高度な役職処理をまだ持たない状態で、基本的な期待勝率計算と画面遷移を成立させる。

- v1 では役職ごとの条件分岐を各処理に直接書く
- v2 以降で roles.py のような役職定義レイヤへ集約を検討する

## 実装対象の役職
- 村
- 占
- 霊
- 狼

補足:
- 占と霊の固有能力判定は v1 では実装しない
- v1 では、占と霊は能力的には素村と同等に扱う
- ただし、レギュレーションとの差異を判定するため、役職グループとしては別物として保持する

## 実装するレギュレーションルール
狼
- 狼の自噛: 無
- 狼の無噛: 無

## 実装する戦略オプション
村
- 処刑対象: `期待勝率最大を選ぶ`
- 村騙りを認めない: `true`
- 対抗が出ない場合は真とみなす: `true`
- ギドラを認める: `false`

狼
- 噛み対象: `期待勝率最大を選ぶ`

占
- 占い能力自体は v1 では実装しない
- ただし、レギュレーションに占を含む場合は、UI 上に `占の戦略オプション` タブだけ表示できるようにする

## 実装する Phase
- 昼
- 夜

## 実装する UI
- メインテーブルの 1 段目は `docs/requirements.md` のカラム案に従う
- 占または霊の可能性がある場合は、中身がダミーの `役職能力タブ` を 2 段目以降に表示する

## 実装順序の案
1. 必要な UI の作成
2. CO ごとにグループ化する仕組みの作成
3. UI への各期待値の反映

補足:
- CO ごとのグループ化がアルゴリズムとして本当に適切かは、実装中に再検証する

## v1 の完成目標
例として `占1 / 霊1 / 村3 / 狼2` のようなレギュレーションに対して、次を満たす。

- 正確な期待値を算出できる
- ラジオボタンで処刑対象を選択し、`Day2`、`Day3` と進んで Tree を辿れる UI がある
- Tree 全体の簡略表示を持ち、戦略オプションどおりに進んだ場合の白陣営 / 黒陣営の勝率と分岐を確認できる

## v1 では未対応だが将来的に見据えるもの
- `ギドラを認める: true` にしたときの複数 CO 対応
- 占と霊の固有能力判定
- 将来的な Phase 拡張
